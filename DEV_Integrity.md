# DEV_Integrity — มาตรฐานความถูกต้องของ Land Lens (Reflex801)

เอกสารชุด DEV Pack หมวด “Integrity” เป็นรากฐานด้านความถูกต้อง แม่นยำ และความสมบูรณ์ของข้อมูลในระบบ Land Lens (LL)

---

## 1. วัตถุประสงค์
- กำหนดกรอบความถูกต้องของข้อมูล LL  
- ป้องกันความคลาดเคลื่อนระหว่าง HL → LL → READY  
- สร้างกติกาที่ทำให้ภาพภาคสนาม “ไม่บิดเบือน” ต้นฉบับ  

---

## 2. ขอบเขตข้อมูลที่ต้องตรวจความถูกต้อง
1. **RAW Images**  
   - ต้องเป็นภาพจากภาคสนามจริง  
   - ห้ามรีทัช / AI enhance ก่อนเข้า HL  

2. **Overlay Layer (LL_Overlay)**  
   - ต้องอ้างอิงตำแหน่งจริง  
   - ข้อความบนภาพต้องเป็นข้อมูลเชิงข้อเท็จจริง  
   - ห้ามสรุปเชิงตีความในขั้นนี้

3. **Annotated Layer (LL_Annotated)**  
   - อนุญาตการวิเคราะห์ แต่ต้องมีหลักฐานภาพรองรับ  

4. **Export Layer (LL_Export)**  
   - เป็นผลลัพธ์สุดท้ายสำหรับ Deck / Report  
   - ต้องผ่าน Checksum ทุกชั้นก่อนส่งออก

---

## 3. กติกาการควบคุมความถูกต้อง (Integrity Rules)
### Rule 01 — “No Synthetic Before HL”
ห้ามภาพใดผ่าน AI ก่อนเข้า HL  
> เหตุผล: ป้องกันสัญญาณเทียม (Synthetic Signal)

### Rule 02 — “Field-first, Interpretation-later”
ขั้น HL / LL ต้องบันทึกอย่างเป็นกลาง ไม่ตีความล่วงหน้า

### Rule 03 — “Traceable Layers”
ทุกภาพต้องย้อนรอยได้ RAW → HL → LL → Export

### Rule 04 — “Locked Metadata”
ข้อมูลต่อไปนี้ต้องไม่ถูกแก้:
- เวลา
- ตำแหน่ง
- อุปกรณ์ถ่ายภาพ

### Rule 05 — “Checksum Validation”
LL Export ต้องมีไฟล์ checksum เช่น `sha256.txt` ทุกครั้งเพื่อใช้ตรวจสอบความสมบูรณ์

---

## 4. โฟลวกระบวนการตรวจสอบความถูกต้อง

1. RAW → ตรวจ metadata  
2. HL_Index → ตรวจข้อมูลเชิงพื้นที่  
3. LL_Overlay → ตรวจตำแหน่งสัญญาณบนภาพ  
4. LL_Annotated → ตรวจเนื้อหาวิเคราะห์  
5. LL_Export → สร้าง checksum  
6. READY → พร้อมใช้งานใน Deck / Report

---

## 5. บทสรุป
DEV_Integrity คือรากฐานที่ทำให้ Reflex801 แตกต่างจากระบบบันทึกภาพทั่วไป เพราะเรายืนยันได้ว่า  
**“ทุกสัญญาณมาจากของจริง — ไม่บิด ไม่แต่ง ไม่ตีความเกินข้อเท็จจริง”**
